# Ivy+ Endowment Performance Analysis (2000–2024)

Comparative analysis of 24 university endowments across four institutional tiers: Ivy League, elite privates, major public universities, and elite liberal arts colleges.

## Project Structure

```
endowment-project/
├── data/
│   ├── endowments.db          # SQLite database (generated by schema.py)
│   ├── raw/                   # Raw source files (990 JSONs, PDFs, financials)
│   │   └── 990s/              # ProPublica API responses + PDF downloads
│   ├── processed/             # Cleaned CSVs for analysis
│   └── sources/               # Downloaded annual reports, financial statements
├── scripts/
│   ├── schema.py              # Initialize database with tables + seed data
│   ├── collect_990.py         # Pull 990 data from ProPublica API
│   └── validate.py            # Data validation + reconciliation checks
├── docs/
│   ├── institutions.md        # Institution selection rationale
│   ├── data_dictionary.md     # Every field documented with sources
│   └── methodology.md         # Ranking framework (to be built)
└── output/                    # Final deliverables (deck, dashboard, exec summary)
```

## Quick Start

```bash
# 1. Initialize the database
python scripts/schema.py

# 2. Collect 990 data (requires internet)
pip install requests --break-system-packages
python scripts/collect_990.py

# 3. Validate data quality
python scripts/validate.py --matrix
```

## Data Sources

- IRS Form 990 (ProPublica Nonprofit Explorer API)
- University audited financial statements
- NACUBO-TIAA Study of Endowments
- SEC Form 13F filings
- University investment office reports
- News coverage and CIO speeches

## Key Design Decisions

1. **Every data point has source provenance** — traceable to a specific document
2. **Confidence levels on all values** — distinguishes reported vs. inferred data
3. **Standardized asset class taxonomy** — maps inconsistent reporting to common categories
4. **Reconciliation log** — tracks and resolves discrepancies between sources
5. **Validation scripts** — automated checks for outliers, coverage gaps, and data quality
