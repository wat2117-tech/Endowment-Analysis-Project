# Working with Uncertainty — Missing-Data Policy

This project treats missing data as information, not as a defect to be
papered over. Coverage is deliberately incomplete: when a number is not
accessible from a citable source, the cell stays NULL and the gap itself is
documented and displayed. This is a methodological choice — an analysis that
distinguishes *known / inferred / unknowable* is more credible than one with
suspiciously complete tables.

## Why data is missing, by category

| Category | Examples | Why inaccessible | Handling |
|---|---|---|---|
| Never published | Most LAC annual returns pre-2015; DUMAC/Rice return series | Small offices publish values, not returns | NULL + note; use NACUBO value anchors only |
| Paywalled / archival | NACUBO tables 2001–2006 for non-flagship schools | Historical studies not freely available | NULL until archives accessed; do not interpolate |
| Entity ambiguity | Berkeley (UC pools most endowment centrally) | "Berkeley's endowment" has no single defensible number | Thin coverage on purpose; requires entity definition before more rows |
| Fiscal-year mismatch | Stanford, Northwestern, UT, Texas A&M (Aug 31 FYE) | Data exists but crisis windows shift two months | Kept, flagged in notes and report caveats |
| Not yet collected | Spending rates, gifts, asset allocation | Requires per-institution PDF work | Empty columns/tables until primary-source pass |

## How the analysis handles gaps

- **No interpolation, no synthetic values.** A missing return-year is simply
  absent from Sharpe/volatility calculations; the `n_years` and `year_span`
  columns disclose exactly what each metric is computed on.
- **Ranking eligibility, not imputation.** Institutions below the minimum
  return-year threshold are listed as *unranked*, not padded into the table.
- **Confidence tiers propagate.** `high` = audited primary document (none
  yet — verification pass pending), `medium` = compiled from published
  reporting, `low` = rough recall flagged for upgrade-or-delete. The
  coverage map and ranking annotations show the mix.
- **Cross-checks over guesses.** Where a second source exists (IRS 990 total
  assets), we reconcile against it (`reconciliation_log`) instead of
  trusting a single figure.

## Artifacts that surface uncertainty

- `output/data_coverage_map.md` — institution × year grid of what exists
  (generated by `scripts/coverage_map.py`)
- `output/phase2_report.md` — caveats section + per-institution data-quality
  annotations
- `python scripts/validate.py --matrix` — live confidence matrix
- `docs/phase1_status.md` — corrections log and remaining-gap inventory
